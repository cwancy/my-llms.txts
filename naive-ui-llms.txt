## OVERVIEW

Naive UI is a Vue 3 component library with 90+ components. It is:
- **Fairly Complete** — 90+ components, all tree-shakable
- **Theme Customizable** — type-safe theming via TypeScript; no CSS/SCSS/Less required
- **TypeScript-Based** — written entirely in TypeScript
- **Fast** — virtual lists on all data-heavy components by default (select, tree, transfer, table, cascader)

No CSS imports are required to use components. Styles are injected via CSS-in-JS (css-render).

Repository: https://github.com/tusen-ai/naive-ui
Documentation: https://www.naiveui.com
Community: https://discord.gg/Pqv7Mev5Dd

## USAGE PATTERNS

### Pattern 1: Direct Import (Recommended — enables tree-shaking)

```vue
<!-- Options API -->
<template>
  <n-button>naive-ui</n-button>
</template>

<script>
import { defineComponent } from 'vue'
import { NButton } from 'naive-ui'
export default defineComponent({
  components: { NButton }
})
</script>
```

```vue
<!-- Composition API with <script setup> -->
<template>
  <n-button>naive-ui</n-button>
</template>

<script setup>
import { NButton } from 'naive-ui'
</script>
```

### Pattern 2: Global Installation (no tree-shaking)

```js
import naive from 'naive-ui'
import { createApp } from 'vue'
const app = createApp(App)
app.use(naive)
// All components available globally as <n-button>, <n-input>, etc.
```

### Pattern 3: Selective Global Installation (preserves tree-shaking)

```js
import { createApp } from 'vue'
import { NButton, NInput, NConfigProvider } from 'naive-ui'
const app = createApp(App)
app.component('NButton', NButton)
app.component('NInput', NInput)
app.component('NConfigProvider', NConfigProvider)
```

### Pattern 4: Auto-Import with unplugin-vue-components (Vite)

```bash
npm i -D unplugin-vue-components unplugin-auto-import
```

```js
// vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import Components from 'unplugin-vue-components/vite'
import AutoImport from 'unplugin-auto-import/vite'
import { NaiveUiResolver } from 'unplugin-vue-components/resolvers'

export default defineConfig({
  plugins: [
    vue(),
    Components({
      resolvers: [NaiveUiResolver()]
    }),
    AutoImport({
      imports: ['vue', 'vue-router'],
      resolvers: [NaiveUiResolver()]
    })
  ]
})
```

### Pattern 5: UMD / CDN

```html
<script src="https://unpkg.com/naive-ui/dist/index.js"></script>
<!-- Available as window.naive -->
```

## CONFIGURATION SYSTEM — NConfigProvider

`NConfigProvider` is the root configuration component. Wrap your entire app (or a subtree) with it to control theme, locale, and namespace.

```vue
<template>
  <n-config-provider :theme="theme" :locale="locale" :date-locale="dateLocale">
    <app />
    <n-global-style />
  </n-config-provider>
</template>
```

### NConfigProvider Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| theme | `Theme \| null \| undefined` | `undefined` | `null` = light theme; `undefined` = inherit from parent; import `darkTheme` for dark |
| theme-overrides | `GlobalThemeOverrides` | `undefined` | Partial override of theme variables |
| locale | `NLocale` | `enUS` | UI locale |
| date-locale | `NDateLocale` | `dateEnUS` | Date locale (for DatePicker etc.) |
| namespace | `string` | `undefined` | CSS namespace; useful to scope detached DOM elements |
| inline-theme-disabled | `boolean` | `false` | Disable inline style theme vars (useful for SSR or cleaner devtools). Not reactive. |
| tag | `string` | `'div'` | Root element tag |
| rtl | `RTLInfo[]` | `undefined` | RTL style overrides |
| breakpoints | `Record<string, number>` | Built-in | Responsive breakpoints |
| abstract | `boolean` | `false` | Render as fragment (no wrapper element) |
| cls-prefix | `string` | `'n'` | CSS class prefix |
| preflight-style-disabled | `boolean` | `false` | Disable preflight (base reset) styles |

### Key Notes on NConfigProvider
- `theme={null}` explicitly forces light theme, ignoring any parent theme.
- `theme={undefined}` (default) inherits from the nearest ancestor `NConfigProvider`.
- Multiple nested `NConfigProvider` instances are supported — inner overrides outer.
- Some parts of components are mounted to `document.body` (e.g. modals, dropdowns). Use `namespace` prop to scope their CSS class.

## THEMING SYSTEM

### Built-in Themes

```js
import { darkTheme, lightTheme } from 'naive-ui'
```

`lightTheme` is the default (components are light themed without any config).
`darkTheme` switches all components to dark mode.

### Applying Dark Theme

```vue
<template>
  <n-config-provider :theme="darkTheme">
    <app />
    <!-- n-global-style syncs theme to document.body (bg color, text color, etc.) -->
    <n-global-style />
  </n-config-provider>
</template>

<script setup>
import { darkTheme } from 'naive-ui'
</script>
```

### OS-Adaptive Theme

```js
import { useOsTheme, darkTheme } from 'naive-ui'
const osTheme = useOsTheme() // Returns Ref<'dark' | 'light' | null>
const theme = computed(() => osTheme.value === 'dark' ? darkTheme : null)
```

### Theme Overrides (Global)

Use `GlobalThemeOverrides` type for full type safety:

```vue
<script lang="ts">
import { NConfigProvider } from 'naive-ui'
import type { GlobalThemeOverrides } from 'naive-ui'

const themeOverrides: GlobalThemeOverrides = {
  common: {
    primaryColor: '#FF0000',
    primaryColorHover: '#FF3333',
    borderRadius: '4px',
    fontWeightStrong: '600'
  },
  Button: {
    textColor: '#FF0000'
  },
  Select: {
    peers: {
      InternalSelection: { textColor: '#FF0000' },
      InternalSelectMenu: { borderRadius: '6px' }
    }
  },
  DataTable: {
    paginationMargin: '40px 0 0 0',
    peers: {
      Empty: { textColor: '#ccc' },
      Pagination: { itemTextColor: '#ccc' }
    }
  }
}
</script>
<template>
  <n-config-provider :theme-overrides="themeOverrides">
    <app />
  </n-config-provider>
</template>
```

### Component-Level Theme Override

Each component also accepts a `theme-overrides` prop for local overrides:

```vue
<n-button :theme-overrides="{ textColor: 'red' }">Red Button</n-button>
```

### Reading Theme Variables in Components

```js
import { useThemeVars } from 'naive-ui'
const themeVars = useThemeVars()
// themeVars.value.primaryColor, themeVars.value.borderRadius, etc.
```

### Theme Editor (Development Tool)

The built-in theme editor must be explicitly imported — it is NOT included in global install:

```vue
import { NThemeEditor } from 'naive-ui'
<n-theme-editor>
  <app />
</n-theme-editor>
```

### Global Style Component

`NGlobalStyle` syncs theme CSS variables to `document.body`. Required for dark mode to affect body background and scrollbar colors:

```vue
<n-config-provider :theme="darkTheme">
  <app />
  <n-global-style />
</n-config-provider>
```

## FONTS

Naive UI works with the optional `vfonts` package:

```js
// main.ts / app entry
import 'vfonts/Lato.css'      // General font
import 'vfonts/FiraCode.css'  // Monospace font
```

If using Lato or OpenSans, configure font weight:

```vue
<n-config-provider :theme-overrides="{ common: { fontWeightStrong: '600' } }">
  <app />
</n-config-provider>
```

To use custom fonts without vfonts, you need `NGlobalStyle`:

```vue
<n-config-provider :theme-overrides="{ common: { fontFamily: 'Inter, sans-serif' } }">
  <app />
  <n-global-style />  <!-- Required for font to respond to theme-overrides -->
</n-config-provider>
```

## SERVER-SIDE RENDERING (SSR)

### Requirements

1. All `@css-render/*` and `css-render` packages in the build must be version `>=0.15.14`.
2. All references to any `@css-render/*` / `css-render` package must resolve to a single version (no duplicates). Check your lock file for `css-render`.

### SSR Setup

```js
// server entry
import { createSSRApp } from 'vue'
import { setup } from '@css-render/vue3-ssr'
import { defineComponent } from 'vue'

const app = createSSRApp(App)
const { collect } = setup(app)

// After renderToString:
const html = await renderToString(app)
const cssText = collect() // Inject this into <head>
```

### SSR with Inline Theme Disabled

Inline theme variables can bloat SSR HTML. Disable them:

```vue
<n-config-provider inline-theme-disabled>
  <app />
</n-config-provider>
```

**Trade-off:** Do not use `inline-theme-disabled` if `theme-overrides` change frequently — stale style elements accumulate.

### Known SSR Limitations

- `n-scrollbar` has SSR bugs in older Vue versions.
- `n-data-table` SSR works from Vue >= 3.2.36.

### Nuxt.js

Install the Nuxt module:
```bash
npm i -D @nuxtjs/naive-ui
```
```js
// nuxt.config.ts
export default defineNuxtConfig({
  modules: ['@nuxtjs/naive-ui']
})
```

## CONTROLLED vs UNCONTROLLED COMPONENTS

Most Naive UI components support both modes:

- **Uncontrolled**: Use `default-value` prop. Component manages its own state.
- **Controlled**: Use `value` + `on-update:value`. Parent owns the state.

```vue
<!-- Uncontrolled -->
<n-input default-value="initial text" />

<!-- Controlled -->
<n-input :value="val" @update:value="val = $event" />
<!-- or with v-model -->
<n-input v-model:value="val" />
```

## COMPOSABLES (PROGRAMMATIC APIs)

All provider-based composables must be called inside a component that is a descendant of the corresponding Provider:

| Composable | Required Provider | Usage |
|-----------|-----------------|-------|
| `useMessage()` | `NMessageProvider` | Show toast messages |
| `useDialog()` | `NDialogProvider` | Show confirm/alert dialogs |
| `useNotification()` | `NNotificationProvider` | Show notification alerts |
| `useLoadingBar()` | `NLoadingBarProvider` | Control top loading bar |
| `useThemeVars()` | `NConfigProvider` | Read current theme variables |
| `useOsTheme()` | None | Detect OS light/dark preference |

### Setting Up All Providers

```vue
<!-- App.vue -->
<template>
  <n-config-provider :theme="theme" :locale="locale" :date-locale="dateLocale">
    <n-loading-bar-provider>
      <n-dialog-provider>
        <n-notification-provider>
          <n-message-provider>
            <router-view />
            <n-global-style />
          </n-message-provider>
        </n-notification-provider>
      </n-dialog-provider>
    </n-loading-bar-provider>
  </n-config-provider>
</template>
```

## KNOWN COMMON ISSUES

### useMessage / useDialog / useNotification returns undefined

These composables only work inside components that are **descendants** of their respective Provider. They cannot be called at the app root level or outside Vue's component tree.

**Solution**: Wrap `<router-view />` (or `<app />`) with the relevant providers, not the other way around.

### Components Appear Without Styles

This usually happens when:
1. Using SSR without proper css-render setup
2. Using `app.use(naive)` with a very old bundler — no CSS is emitted, which is intentional. Styles are injected at runtime.

### Virtual Scroll / Large Data

For `NDataTable`, `NTree`, `NSelect`, `NCascader`, `NTransfer` — virtual scrolling is available but usually must be opted into via `virtual-scroll` prop. `max-height` is required for `NDataTable` virtual scroll.

### NDataTable Remote Mode

Set `remote: true` when handling pagination, filtering, and sorting server-side. You must manually update `data` in response to `on-update:page`, `on-update:filters`, and `on-update:sorter` events.

### Form Validation with Number Types

When validating number fields with `required`, add `type: 'number'` to the rule:

```js
const rules = {
  age: { type: 'number', required: true, message: 'Required', trigger: 'blur' }
}
```

### Multiple Providers for Different Subtrees

You can nest `NConfigProvider` to scope themes:

```vue
<n-config-provider :theme="null">          <!-- Light for most of app -->
  <main-content />
  <n-config-provider :theme="darkTheme">   <!-- Dark sidebar only -->
    <sidebar />
  </n-config-provider>
</n-config-provider>
```
